# 批量视频上传 · 流量舱 PublishFlow

[![License: MIT](https://img.shields.io/badge/License-MIT-6C5CE7.svg)](./LICENSE)

面向 Windows 10 / 11 的多平台视频发布工作台原型。流量舱把视频整理、基础信息填写、平台适配、账号管理和发布追踪集中在一个清晰的工作台里，让创作者一次准备内容，再顺畅接力到抖音、快手与哔哩哔哩官方发布页面。当前版本采用纯前端实现，开箱即可预览，不采集账号密码。

> 一次准备，处处发布。让多平台内容运营从重复劳动，变成清晰、可控的一条工作流。

## 下载 Windows 桌面版

前往 [GitHub Releases](https://github.com/QXF19/qxf-video/releases) 下载最新版 `批量视频上传-Setup-*-x64.exe`。安装包支持 64 位 Windows 10 与 Windows 11，安装后会创建开始菜单和桌面快捷方式。

桌面版采用独立窗口、原生窗口控制、单实例运行和安全隔离设计。由于当前发行版尚未购买商业代码签名证书，Windows SmartScreen 可能显示“未知发布者”，可在确认下载来源为本仓库后继续安装。

## 一键启动

双击 `一键启动.cmd`。脚本会在本机启动一个只监听 `127.0.0.1` 的静态服务，并使用 Microsoft Edge 应用模式打开，视觉上与独立桌面应用一致。启动器会自动选择 Node.js 或 Python；如果都未安装，会直接打开离线界面。

也可以直接打开 `index.html`，但离线缓存等能力需要通过一键启动使用。

## 接入真实发布能力

当前扫码为演示流程；“官方发布接力”会复制整理好的内容并打开平台官方上传页，但不会绕过验证码、代选本地文件或替用户点击最终发布。生产版本应通过各平台官方开放平台完成：

1. 创建企业/开发者应用并获得 `client_id` 与回调地址。
2. 服务端实现 OAuth / 扫码授权，令牌必须加密存储，不能放在前端代码中。
3. 分别接入平台提供的视频上传、分片、稿件提交和发布状态查询接口。
4. 后端增加任务队列、失败重试、限流、日志审计与令牌刷新。
5. 发布前明确展示每个平台的标题、封面、话题和可见范围，避免误发。

建议生产架构：桌面端使用 Tauri（Windows 10 1809 及以上、Windows 11），服务端使用 Node.js/NestJS 或 Java/Spring Boot，任务队列使用 Redis/BullMQ。平台能力与资质要求会变化，应以各平台最新开放平台文档为准。
